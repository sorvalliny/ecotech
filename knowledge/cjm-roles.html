Customer Journey Map по ролям

Путь каждой роли в ОРБИТА — от первого входа до ежедневной работы. Источник правды для развития платформы.

CEO BU / Руководитель направления
Цель: за 30 секунд понять состояние портфеля. Решения на комитетах. Контроль рисков без запросов у команд.

Ежедневный путь

1
Открыть каталог продуктов
Все продукты, стадии, статусы — на одном экране. Светофор показывает кто в зелёной зоне, кто в красной.
catalog.html
2
Проверить проблемные продукты
Клик по продукту с жёлтым/красным статусом → карточка с деталями: что случилось, кто блокирует, когда решение.
workspace.html
3
Посмотреть лог изменений
Что изменилось с прошлого раза: Gate решения, новые инициативы, смены статусов.
каталог → лог

Еженедельный путь

4
Ревью инициатив (Go / No-Go / Revise)
В карточке продукта → таб "Инициативы". Открыть инициативу → оставить резолюцию.
workspace → инициативы
Частота: 1 раз в неделю или по запросу PMO

Ежемесячный путь

5
Gate Review комитет
Рассмотрение продуктов на переход между стадиями. Чеклист артефактов, решение Go/No-Go, протокол.
committees.html
Частота: 1 раз в месяц

Product Manager (владелец продукта)
Цель: управлять продуктом из одного места. Планировать, приоритизировать, каскадировать на команду. Проходить Gate Review.

Ежедневный путь

1
Открыть свой продукт
Главная → "Мои продукты" → клик. Или прямая ссылка workspace.html?id=ECO-XX.
workspace.html
2
Обновить статус (светофор)
Паспорт → статус-карточка. Выбрать зелёный/жёлтый/красный, написать что происходит.
Частота: 1 раз в неделю (минимум)
3
Работать с инициативами
Таб "Инициативы" → создать новую, обновить RICE, сменить статус, добавить ссылку на YouTrack.
workspace → инициативы

Еженедельный путь

4
Синк с PMO
Обсудить блокеры, статус инициатив, подготовку к комитету. PMO координирует.
Частота: 1 раз в неделю

Квартальный путь

5
Планирование квартала
Таб "Цели и планирование" → поставить цели H, декомпозировать в инициативы Q, оценить RICE, согласовать с PMO.
workspace → цели
6
Подготовка к Gate Review
Проверить чеклист артефактов. Заполнить недостающие. Запросить ревью через платформу.
gate-checklist.html

При запуске нового продукта

7
Подать заявку (бриф)
Заполнить бриф с описанием проблемы, ЦА, RICE-оценкой. Попадает в реестр идей → скрининг PMO.
brief.html

Участник команды (Member)
Цель: знать свои инициативы, понимать приоритеты, связать работу с трекером задач.

Ежедневный путь

1
Открыть карточку продукта
Видит свои инициативы (по скоупу назначения). Статус, RICE, ссылка на трекер.
workspace → инициативы
2
Обновить статус инициативы
Сменить статус (planned → wip → done). Обновить описание или RICE если изменились вводные.
3
Перейти в трекер задач
По ссылке из инициативы → YouTrack/Shiva. Там эпики и задачи. ОРБИТА = стратегический уровень, трекер = операционный.

Боли

Не понимаю приоритет — что делать первым?
Нет связи между стратегией (цели) и моими задачами

1 клик
