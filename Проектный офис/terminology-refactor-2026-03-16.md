## ФИНАЛЬНАЯ МОДЕЛЬ

### Иерархия сущностей

```
ПРОДУКТ (постоянный) — долгоживущая сущность в портфеле
│  Имеет команду, пользователей, метрики, Орбиту (О·0–О·6)
│
└─ ЦЕЛЬ (полугодие/год) — OKR: Objective + Key Results
   │
   └─ ИНИЦИАТИВА (квартал/полугодие) — крупная единица работы
      │  Бывший «проект». Временная, с дедлайном и результатом.
      │
      └─ ЭПИК (3–6 недель) — часть инициативы, один deliverable
         │
         └─ STORY → ЗАДАЧА (дни)
```

### Стадии (Орбиты) — где продукт

```
О·0  Заявка        (бывш. Идея)
О·1  Разведка       (бывш. Discovery)
О·2  Прототип       (бывш. Пилот)
О·3  Разгон         (бывш. Запуск)
О·4  Экосистема     (бывш. Масштаб)
О·5  Плато          (бывш. Зрелость)
О·6  Архив          (бывш. Консервация)
```

### Фазы витка — что делает команда

```
Исследование → Разработка → Оценка и ретро
```

### Правило: «Проект» как сущность упразднён

| Контекст | Решение |
|----------|---------|
| `project` в коде | → `product` |
| «Проектный офис» | Оставить (устоявшееся название PMO) |
| «Трекер проектов» | → «Трекер продуктов и инициатив» |
| Кто-то говорит «проект» | = Инициатива (зафиксировать в глоссарии) |
