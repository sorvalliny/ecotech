Развилки

Развилка 1: после исследования

  • Идея простая → передать в коммерцию (департамент = менеджер проекта)
  • Идея требует MVP → делаем сами, проходим комитеты

Развилка 2: градация MVP

  • Высокий репутационный риск → полноценный прототип + аудит ИБ
  • Низкий риск → песочница с когортой, лимиты запросов

Развилка 3: после запуска

  • Технологический проект → остаётся в экосистеме
  • Завязан на коммерцию → передаётся, департамент сопровождает

O·3 · WB Книги

O·3→O·5 · Мост поколений

MAU >100К, но нет монетизации. Q3: решение УК — подписка, лицензии или закрытие.

Песочница MVP и ядро команды

Песочница MVP

Изолированная среда с лимитами RPS, когортой 100-1000 человек, без полного аудита ИБ. Полный аудит — только перед O·3 (выход на рынок).

Ядро команды > внешние модули

Фиксированная команда заточена на быстрые MVP. Внешние модули — исключение через решение УК. Эксперты коммерции — как консультанты, не в штат.

Управление и принятие решений

Три комитета — органы принятия решений

Каждый комитет ведёт тот блок CORE, который за него отвечает. Руководитель блока организует повестку, готовит материалы, ведёт протокол и контролирует исполнение.

Финансовый комитет

Цель: стоит ли строить? Есть ли рынок и деньги

Частота: еженедельно

Состав: директор + инвест-директор + рук. «бизнес и финансы» + эксперт коммерции

Стадии: Заявка → Разведка → Разгон

→

Технический комитет

Цель: можно ли построить? Безопасно ли

Частота: после финансового, по готовности

Состав: рук. техблока + ИБ + техлид проекта

Стадии: Разведка → Прототип → Экосистема

→

Управляющий комитет

Цель: финальное решение — запускаем, останавливаем, перенаправляем

Частота: еженедельно

Состав: CEO + директор департамента + инвест-директор

Стадии: все (от заявки до зрелости)

Ресурсная модель и запрос

Фиксированная команда + решение комитета

Штат не раздуваем. Новый человек — только через обоснование и решение управляющего комитета.

Основной ресурс — фиксированная команда

  • На старте утверждается штат с грейдами
  • Каждый квартал директор выбирает 3-5 приоритетов
  • Рук. блоков оценивают: сколько людей на проект
  • Стоимость проекта = грейды × срок. Прозрачно для CEO
  • Если людей не хватает — проект ждёт следующий квартал

Дополнительный ресурс — когда ждать нельзя

  • Руководитель блока готовит обоснование
  • Обоснование выносится на управляющий комитет
  • Одобрили → людей на конкретный проект и срок
  • Не одобрили → приоритеты внутри текущей команды
  • Внешние эксперты — через запрос, не в штат

Что просим

Два параллельных стрима: собственные продукты + дистрибуция партнёров

CORE-команда из 4 блоков работает на оба стрима. Стрим 1 — длинный цикл, высокая маржа. Стрим 2 — быстрые деньги и валидация ниш за 60 дней. Три блока укомплектованы, один — вакансия. Через 90 дней — первые транзакции.
