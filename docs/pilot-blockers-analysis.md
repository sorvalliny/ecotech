# Критические задачи — что блокирует пилот
## Анализ COO для Виктора | 17 марта 2026

## РЕАЛЬНО БЛОКИРУЕТ (без этого нельзя запускать)

### 1. Shared storage (V-02) — команды не видят общие данные
- **Проблема:** localStorage = один браузер. PM заполнил анкету — PMO не видит.
- **Влияние:** На пилоте PMO вводит данные сам → команды смотрят на демо. Работает, но не масштабируется.
- **Решение для пилота:** PMO вводит данные при команде на синке. Экспорт JSON через админку для бэкапа.
- **Вердикт:** НЕ блокер для управляемого пилота (2–3 команды, PMO ведёт). Блокер для self-service.

### 2. 49 MEDIUM/LOW багов (V-01 остаток)
- **Проблема:** мелкие UI-баги, текстовые ошибки, недостающие валидации.
- **Влияние:** команда может наткнуться на баг и потерять доверие к платформе.
- **Решение:** прогнать ТОП-10 самых заметных, остальные — после пилота.
- **Вердикт:** ЧАСТИЧНЫЙ блокер. Нужно исправить ТОП-10 перед встречей.

### 3. Валидация изменений (T-10) — нет workflow утверждения
- **Проблема:** любой PM может изменить любой продукт. Нет апрувов.
- **Влияние:** На пилоте (2–3 команды) — управляется вручную. В масштабе — хаос.
- **Вердикт:** НЕ блокер для пилота. Блокер для масштабирования.

## ВАЖНО, НО НЕ БЛОКИРУЕТ ПИЛОТ

### 4. Живые шаблоны (S-03)
- Шаблоны = текст, не формы. PM копирует и заполняет вручную.
- **Для пилота:** достаточно. PM привыкли к Google Docs.

### 5. Витрина для бизнеса (B-01)
- business.html не показывает живые данные из реестра (частично работает).
- **Для пилота:** CEO не смотрит business.html ежедневно. Показать на статусе вручную.

### 6. Кросс-зависимости (B-013)
- Нет инструмента для 2–3 команд.
- **Для пилота:** управляется на синке устно.

## ВЕРДИКТ COO

**Пилот можно запускать в формате «управляемый PMO»:**
- PMO вводит данные
- Команды смотрят и валидируют на синках
- Баги фиксим параллельно
- Self-service — после shared storage + валидация

**Рекомендую:**
1. Исправить ТОП-10 багов из V-01 (0.5 ИИ-часа)
2. Запустить установочную встречу в четверг 20.03
3. Первый Gate Review — неделя 3–4 (7–10 апреля)

**Что реально блокирует масштабирование (после пилота):**
- Shared storage (3 дня)
- Серверная авторизация через WB ID (5 дней)
- Миграция на GitLab (5 дней)
