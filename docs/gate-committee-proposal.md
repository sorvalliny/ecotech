# Предложение по составу Gate Review комитета
## Для решения Виктора

---

## Варианты

### Вариант А: Минимальный (для пилота)

| Роль | Кто | Голос |
|------|-----|-------|
| Председатель | Руководитель ПО (Виктор) | Финальное решение |
| Продуктовый эксперт | CPO направления | Go/No-Go |
| Технический эксперт | TL продукта | Техническая готовность |

**3 человека, 40 мин.** Достаточно для Gate 0–2.

### Вариант Б: Расширенный (для Gate 3+)

| Роль | Кто | Голос |
|------|-----|-------|
| Председатель | CEO департамента | Финальное решение |
| Продуктовый | CPO направления | Go/No-Go |
| Технический | CTO / TL | Техготовность |
| Финансовый | CFO / финдиректор | P&L валидация |
| PMO | Руководитель ПО | Процесс, SLA |

**5 человек, 60 мин.** Для Gate 3–4 (инвестиционные решения).

### Вариант В: Адаптивный

| Gate | Кто решает | Формат |
|------|-----------|--------|
| Gate 0 | PMO (Виктор) | Async, 5 дней |
| Gate 1 | PMO + CPO | 30 мин |
| Gate 2 | PMO + CPO + TL | 40 мин |
| Gate 3 | CEO + CPO + CFO | 60 мин |
| Gate 4–5 | Инвест. комитет | По регламенту |
| Gate 6 | PMO | Чеклист |

---

## Рекомендация COO

Для пилота — **Вариант В**. Причины:
- Gate 0 пилотных продуктов проводит Виктор единолично (быстро, без бюрократии)
- Gate 1–2 добавляем CPO и TL (нужна экспертиза)
- Gate 3+ — когда дойдём, расширим состав

**Нужно решение:**
1. Согласны с вариантом В?
2. Кто CPO для каждого пилотного продукта? (может быть один на все три)
3. Нужно ли приглашать кого-то дополнительно на установочную встречу?
