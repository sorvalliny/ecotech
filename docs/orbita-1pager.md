# ОРБИТА — Операционная система продуктового портфеля

## Проблема

У 10+ продуктовых команд нет единого языка: приоритеты назначаются по голосу руководителя, каждая команда ведёт бэклог в своём формате, стейкхолдеры получают статус только по запросу. Результат — дублирование, потерянные инициативы, решения без данных.

## Решение

ОРБИТА — единый фреймворк управления продуктовым портфелем: Gate-модель из 7 стадий, RICE-приоритизация, реестр с прозрачным статусом каждого продукта. Комитет принимает решения Go/No-Go по чеклисту, а не по ощущениям.

## Как работает

```
Заявка → Разведка → Прототип → Разгон → Экосистема → Плато
 (G0)     (G1)       (G2)      (G3)       (G4)        (G5)
```

1. PM подаёт заявку с RICE-оценкой — PMO проводит скрининг за 5 дней — Gate 0
2. Команда проходит витки (Исследование — Разработка — Оценка) внутри каждой стадии
3. Gate Review — комитет решает Go / No-Go / Pivot по чеклисту обязательных артефактов

## Что в системе сейчас

- 3 продукта в пилоте (Маркетплейс К, Генератор О, Сервис З)
- 7 инициатив с RICE-оценкой
- 3 цели с Key Results
- Реестр с 4 видами (Cards / Table / Kanban / История)
- Gate-чеклист по 7 стадиям (G0-G6)
- Авторизация с 4 ролями (CEO, PM, Product Lead, PMO)

## Для руководителя

- Портфель продуктов в одном месте — без запроса PM
- Статус каждого продукта: стадия, метрики, риски, блокеры
- Gate-воронка: сколько продуктов на каждой стадии, кто застрял
- Решения на Gate Review подкреплены данными, не мнениями
