3. Онбординг команд
1Обучение PMO1 день
  • Знакомство с платформой: реестр, карточка продукта, фреймворк
  • Настройка прав, создание продуктов, назначение PM
2Обучение PM2 дня
  • Работа с карточкой продукта: паспорт, цели, инициативы
  • Заполнение брифа, RICE-оценка, планирование Q/H
  • Связь с внешним трекером (YouTrack)
3Обучение Lead / Заказчик0.5 дня
  • Как смотреть портфель, как делать ревью (Go / No-Go)
  • Как читать метрики и статусы
4Пробный прогон1 неделя
  • 1-2 продукта проходят полный цикл O·0 → O·1
  • Сбор обратной связи, корректировка процессов

Фаза 1: Подготовка (неделя 1-2)
  • Критические доработки платформы
  • Деплой на корп. Git
  • Назначение ролей
  • Онбординг PMO
Фаза 2: Пилот (неделя 3-6)
  • 2-3 продукта через O·0 → O·1
  • SSO интеграция параллельно
  • Еженедельный сбор фидбека
  • Первый Gate Review комитет
Фаза 3: Стабилизация (неделя 7-8)
  • Backend API (замена localStorage)
  • Корректировка процессов по фидбеку
  • Онбординг оставшихся PM
Фаза 4: Масштабирование (неделя 9-12)
  • Все продукты портфеля в ОРБИТА
  • Полный цикл планирования H2 2026
  • Регулярные Gate Review комитеты
  • Интеграция с YouTrack (если приоритет)
Фаза 5: Зрелость (месяц 4+)
  • 100% продуктов с актуальными данными
  • AI-инструменты интегрированы (SM-003)
  • Квартальные отчёты эффекта
