Что нужно настроить для Обучения
  • 1 Инвентаризация портфеля — какие продукты и инициативы есть в департаменте Обучения? Занести в реестр ОРБИТА.
  • 2 Назначение ролей — PMO-менеджер (Обучение), PM для каждого продукта, Заказчики.
  • 3 Адаптация процессов — какие стадии Gate актуальны? Нужны ли специфические артефакты для образовательных продуктов?
  • 4 Планирование — цели H1 2026 по каждому продукту, декомпозиция в инициативы Q2.
  • 5 Пилот — выбрать 1–2 продукта для полного прохождения через ОРБИТА.
Специфика образовательных продуктов
•
Контент как продукт
Курсы, учебные материалы, методики. Возможно нужен свой тип сущности или процесс в реестре.
•
Аккредитация / лицензирование
Дополнительные сопроводительные процессы — вместо ТЗ могут понадобиться образовательные стандарты.
•
Сезонность
Учебные циклы, наборы, выпуски. Планирование привязано к учебному году, а не только к кварталам.
•
Метрики
Completion Rate, NPS обучения, Time to Competency — вместо DAU/Retention.
•
Заказчики
HR, руководители подразделений, внешние клиенты — другой профиль стейкхолдеров.
Формат совместной работы
Вариант A
Полное сопровождение
Проектный офис ведёт ОРБИТА для обоих департаментов. Единый PMO Lead, единые стандарты, общий ритм планирования и отчётности.
Вариант B
Консалтинг + передача
Проектный офис настраивает ОРБИТА для Обучения, обучает PMO-менеджера, передаёт в самостоятельное управление департаменту.
