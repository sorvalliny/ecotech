# B-005 | Трекер продуктов и инициатив
## Агент: Разработчик платформы | Приоритет: P1

---

## Цель

Создать `Проектный офис/tracker.html` — единый интерфейс для управления портфелем продуктов и инициатив с Gate-фильтрацией.

---

## Функциональные требования

### Вкладка «Пилот»
- Фильтр: только продукты с `pilot: true` из data.json
- Карточка продукта: название, текущая Орбита (О·0–О·6), статус (on-track/at-risk/blocked)
- Индикатор: ближайший чекпоинт (дата), блокер (если есть)
- Цветовое кодирование: зелёный/жёлтый/красный по статусу

### Вкладка «Портфель»
- Все продукты из ECOTECH_PRODUCTS
- Фильтры: по Орбите (О·0–О·6), по статусу, по типу (B2C/B2B/B2G)
- Колонки: Название, Орбита, RICE Score (цветовая зона), Владелец, Обновлено
- Сортировка: по RICE Score (desc), по Орбите, по дате

### Вкладка «Инициативы»
- Список из ECOTECH_BACKLOG
- Группировка по продукту
- RICE Score + фаза (Исследование/Разработка/Оценка)
- Статус: Бэклог / Запланировано / В работе / Готово

### Admin-режим
- Переключатель (toggle) в header
- Inline-редактирование: клик по полю → редактирование → сохранение
- Modal для добавления нового продукта / инициативы
- Данные сохраняются через `window.PT.save()`

---

## Технические требования

- Данные: `window.PT.load(PT.KEY_PRODUCTS)`, `window.PT.load(PT.KEY_BACKLOG)`
- Стек: HTML + CSS + Vanilla JS (как rice.html, templates.html)
- Подключить: icons.js, platform.js, nav.js
- Дизайн-система: токены из :root, шрифты Orbitron/Exo2
- Без emoji — только SVG из icons.js
- Responsive: работает на 1024px+ (desktop first)
