# CPO UX Review -- Прогон пилотных flow

> Дата: 2026-03-17
> Роль: CPO, глазами PM первый визит
> Метод: walkthrough каждого flow по HTML-коду, анализ структуры, навигации, CTA, состояний

---

## FLOW-1 | Первый визит (product.html)

**Статус: Хорошо**

### Что работает
- Hero-секция четко говорит "Сделай следующий шаг по продукту" -- PM понимает, что это его точка входа
- Два CTA в hero: "Подать заявку на продукт" (primary) и "Впервые здесь?" (secondary) -- хорошее разделение новичков и опытных
- Орбитальная визуализация -- кликабельные планеты ведут на Brief, Фреймворк, Трекер, Стратегию, Шаблоны, Продукт
- Секция "Быстрые действия" -- 6 карточек-действий с понятными названиями и описаниями
- Онбординг-баннер внизу "Впервые на платформе?" -- страховочная сетка для тех, кто промотал hero
- Skip-link для accessibility

### Что путает
- Орбитальная визуализация на мобильных (< 520px) скрыта (`display:none`) -- PM теряет визуальную навигацию, остаются только текстовые карточки ниже
- CTA "Подать заявку на продукт" ведет на `brief.html`, но action-card тоже "Подать заявку на продукт" ведет туда же -- дублирование, PM может думать что это разные страницы
- Карточка "Карточка продукта" (featured) ведет на `tracker.html#tab=portfolio` -- но PM, который еще не создал продукт, получит пустой трекер
- Нет явной индикации "где я сейчас" в навигации (nav.js генерирует навбар, но active state не виден в product.html)

### Навигация за 3 клика
- Трекер: 1 клик (карточка "Проверить статус в трекере" или планета "Трекер")
- Фреймворк: 1 клик (карточка "Изучить фреймворк G0-G6" или планета "Фреймворк")
- Шаблоны: 1 клик (карточка "Взять шаблон артефакта" или планета "Шаблоны")
- Все доступно за 1 клик -- отлично

### Рекомендации
1. **Featured-карточка "Карточка продукта"**: добавить состояние empty state -- если у PM нет продукта, показывать подсказку "Сначала подайте заявку"
2. **Мобильная навигация**: при скрытии орбиты добавить компактный grid-навигатор (иконки + label) вместо полного скрытия
3. **Hero CTA**: переименовать secondary из "Впервые здесь?" в "Как это работает?" -- менее стигматизирующе
4. **Action card Brief**: в action-go написано "Открыть RICE", хотя карточка "Подать заявку на продукт" -- несоответствие copy, PM путается что такое RICE

## FLOW-2 | Трекер (tracker.html)

**Статус: Проблемы**

### Что работает
- Toolbar с четырьмя табами: Продукты / Инициативы / Цели / История -- понятная структура
- Кнопка "+ Создать" с выпадающим меню (Продукт / Цель / Инициативу) -- PM видит все действия
- Context-bar: выбор продукта (select), фильтры, поиск, переключатель view (Cards/Kanban)
- Onboarding overlay (кнопка "?" в toolbar) -- 5-шаговый тур, можно пропустить
- Edit panel справа (drawer) с табами для детального редактирования инициативы
- Kanban view для Продуктов и Инициатив
- Gate-прогресс бар на карточках продуктов (визуальные точки done/current/todo)

### Что путает
- **Title страницы "RWB - Product OS"** -- не совпадает с брендом "ОРБИТА" из product.html; PM может подумать что попал на другую платформу
- **Context-bar "Все продукты"**: при первом визите select пуст (options генерируются из JS/localStorage) -- PM видит пустой dropdown и не понимает что делать
- **View Switch**: кнопки Cards/Kanban рендерятся в `#active-views` через JS, но контейнер `view-switch` находится внутри `ctx-right` с `margin-bottom:14px` -- визуально отрывается от контента
- **Sidebar скрыт**: CSS `.sidebar{display:none !important}` -- фильтры по типу инициативы (Исследование/Разработка) существуют в HTML но недоступны пользователю
- **"+ Создать" > "Инициативу"**: вызывает `addInitiative(contextProduct||'')` -- если контекст пуст, инициатива создается без привязки к продукту, что позже путает
- **Нет breadcrumbs**: PM не видит путь обратно на product.html (nav.js подгружает breadcrumbs.js, но нет явного элемента)
- **Deep link `#tab=portfolio`**: работает через JS парсинг hash, но при прямом переходе product.html -> tracker.html#tab=portfolio может быть race condition (данные еще не загрузились)

### Рекомендации
1. **Унифицировать title**: заменить "RWB - Product OS" на "ОРБИТА - Трекер" для единообразия бренда
2. **Empty state трекера**: при пустом списке продуктов показать hero-карточку "Создайте первый продукт" с кнопкой, ведущей на brief.html
3. **Раскрыть sidebar или перенести фильтры**: sidebar с `display:none !important` делает фильтры по типу инициативы недоступными; перенести в context-bar или убрать из DOM
4. **Context select**: при отсутствии продуктов показывать placeholder "Нет продуктов -- создайте первый"
5. **Защита "+ Создать Инициативу"**: если нет продукта, показывать tooltip "Сначала создайте продукт"
6. **Breadcrumbs**: добавить явную навигационную цепочку "ОРБИТА > Трекер > [таб]"

## FLOW-3 | Gate-чеклист (gate-checklist.html)

**Статус: Хорошо (с замечаниями)**

### Что работает
- Заголовок "Gate-чеклист" с иконкой -- сразу понятно назначение
- Select "Выберите продукт" -- прямой выбор контекста
- Табы Gate 0...Gate 6 генерируются динамически -- PM видит все стадии
- Каждый Gate: информация о переходе, кто принимает решение, SLA
- Прогресс-бар: визуальный индикатор заполненности чеклиста
- Чеклист разделен на "Обязательные" и "Желательные" (must/nice) -- приоритизация
- Чекбоксы кликабельные, с анимацией, состояние сохраняется в localStorage
- Кнопка "Запросить Gate Review" с disable-логикой (пока не заполнены must-пункты)
- Deep link через hash: `#gate=2&product=xxx` -- прямой переход из трекера/фреймворка работает
- Ссылки на шаблоны и связанные инструменты в чеклисте (brief.html, strategy.html)

### Что путает
- **Обратная ссылка**: `back-link` ведет на `tools.html`, а не на трекер -- PM, пришедший из трекера, ожидает вернуться обратно
- **Select продуктов пуст при первом визите**: как и в трекере, если данных нет в localStorage -- PM видит пустой dropdown и placeholder "Выберите продукт и Gate"
- **Gate Review**: кнопка генерирует `mailto:` на `sorval.viktor@rwb.ru` -- это жесткий хардкод, не масштабируется; PM может не понять что "запрос" -- это письмо
- **Нет связи с трекером**: после заполнения чеклиста данные остаются только в localStorage, не обновляют Gate-статус продукта в трекере

### Рекомендации
1. **Back-link**: определять referrer или передавать `?from=tracker` -- и показывать "Назад в трекер" вместо "Инструменты"
2. **Empty state**: при пустом select показать кнопку "Перейти к созданию продукта" -> brief.html
3. **Gate Review**: заменить mailto на форму/модалку с подтверждением + добавить visual feedback "Заявка отправлена"
4. **Синхронизация с трекером**: при заполнении чеклиста на 100% (must) автоматически обновлять Gate-статус продукта в общем хранилище (platform.js)
5. **Progress feedback**: добавить toast или badge "Осталось N обязательных пунктов" при частичном заполнении

## FLOW-4 | Фреймворк (framework-lite.html, секция sec-s2)

**Статус: Хорошо**

### Что работает
- Секция О-2 "Прототип -- Разработка": четкий заголовок, tagline, duration badge (6-12 недель)
- Три колонки: Процесс (нумерованный список), Артефакты (список с маркерами), Инструменты (список со стрелками) -- PM видит full picture
- Gate-карточка: критерии перехода Gate 2 -> О-3, кто принимает, SLA, формат, шаблоны
- Три трека (Бизнес / Продукт / Разработка) -- цветовое кодирование, PM видит свой трек
- "Виток" -- визуализация итерационного цикла (Исследование -> Разработка -> Оценка и ретро) с таймингами
- **Быстрые кнопки** внизу каждой стадии:
  - "+ Создать инициативу" -> tracker.html#tab=backlog
  - "Gate-чеклист" -> gate-checklist.html#gate=2
  - "Открыть трекер" -> tracker.html#tab=portfolio
- Кнопки есть на КАЖДОЙ стадии (О-0 -- О-6) с корректными gate-номерами -- отлично
- Шаблоны открываются через `openTmpl()` прямо из gate-карточки

### Что путает
- **Кнопка "+ Создать инициативу"** ведет на `tracker.html#tab=backlog` -- но не передает контекст стадии; PM ожидает что инициатива будет привязана к текущему Gate
- **Три трека** не кликабельные -- PM не может перейти к детальному описанию трека, это только readonly информация
- **Шаблоны**: `openTmpl('prd')` -- если функция не подгрузилась (races), клик ничего не делает; нет fallback
- **Навигация между стадиями**: нет sticky TOC или стрелок "предыдущая/следующая стадия" -- PM на длинной странице теряется

### Рекомендации
1. **Передавать контекст стадии**: ссылка "+ Создать инициативу" -> `tracker.html#tab=backlog&stage=2` -- трекер предзаполняет стадию
2. **Sticky навигация стадий**: добавить floating sidebar или sticky pills (О-0 ... О-6) для быстрого перемещения по фреймворку
3. **Шаблоны fallback**: если `openTmpl` не загружен, делать навигацию на templates.html#tpl=prd
4. **Треки кликабельные**: добавить tooltip или expandable content с деталями каждого трека

## Общая оценка UX-зрелости пилота

Платформа демонстрирует **хорошую информационную архитектуру**: PM находит трекер, фреймворк и шаблоны за 1 клик с главной. Gate-модель хорошо визуализирована. Основные проблемы -- в **состояниях при первом визите** (empty states) и **согласованности** между модулями (бренд, данные чеклистов, обратная навигация). После исправления P0-проблем платформа готова к пилотному запуску.
