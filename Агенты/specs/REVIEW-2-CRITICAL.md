- **Проблема:** Демо называется "💡 ОРБИТА Демо" — непонятно. Должен быть полноценный пример.
- **Фикс:** Переименовать DEMO в реальный продукт-пример с полными данными, онбордингом для ролей Наблюдатель и Менеджер.

## БЛОК 2: АВТОРИЗАЦИЯ И ПРАВА (БЛОКЕР)

### RC-04 | Авторизация сырая — нет логина/пароля
- **Проблема:** Любой может зайти в админку. Нужен вход по корп почте @rwb.ru + пароль на почту.
- **Фикс:** Форма входа (email @rwb.ru), генерация OTP, проверка. На localStorage MVP — хранить hash email + session token.

### RC-05 | Админ видит то же что гость
- **Проблема:** Нет панели управления доступами. Админ не может выдать права.
- **Фикс:** В admin.html: таблица пользователей → назначение роли + продуктов → сохранение.

### RC-06 | Нет разделения Наблюдатель / Менеджер в трекере
- **Проблема:** Все видят и могут всё. Наблюдатель должен видеть мониторинг без редактирования. Менеджер — редактировать свой продукт.
- **Фикс:** Проверка OrbAuth.canEditProduct(productId) перед каждым действием. Скрытие кнопок для viewer.

### RC-07 | Окно авторизации при входе в трекер
- **Проблема:** Трекер открывается без авторизации.
- **Фикс:** auth-ui.js уже показывает модал — проверить что работает на tracker.html.

## БЛОК 3: НАВИГАЦИЯ И АРХИТЕКТУРА (ВЫСОКАЯ)

### RC-08 | Хлебные крошки пустые / с названиями папок
- **Проблема:** "Проектный офис" / "Product OS" — рудименты.
- **Фикс:** Переименовать breadcrumbs.js маппинг. Убрать кириллические папки из отображения.

### RC-09 | Трекер = плохое название (путает с YouTrack)
- **Проблема:** Пользователи путают "трекер продуктов ОРБИТА" с "трекером задач YouTrack".
- **Фикс:** Переименовать. Варианты: "Реестр", "Портфель", "Командный мост", "Панель управления". Рекомендация: **"Реестр"** — нейтрально, понятно.

### RC-10 | Витрина продуктов → отдельный раздел в навигации
- **Проблема:** Витрина спрятана в business.html. Нужна отдельно: над «Статус портфеля».
- **Фикс:** business.html = витрина (ребренд), dashboard.html = статус. Навигация: Витрина → Статус → Реестр.

### RC-11 | Рудиментарные кнопки «добавить» в трекере
- **Проблема:** Остались отдельные кнопки при наличии единой «+ Создать».
- **Фикс:** Удалить все отдельные кнопки добавления кроме «+ Создать» в toolbar.

### RC-12 | Подать заявку = логика «Бизнесу», не «Продукту»
- **Проблема:** brief.html на странице Продукту. Должна быть в логике Бизнесу — подача брифа.
- **Фикс:** Перенести brief.html в корень или Бизнесу/. CTA на business.html. Убрать из product.html.

### RC-13 | При отправке брифа — попадает в никуда
- **Проблема:** Brief создаёт запись в BACKLOG, но не как продукт на начальной стадии.
- **Фикс:** submitBrief() → создать продукт в ECOTECH_PRODUCTS (gate:0, status:'pending') + запись в BACKLOG. Видимость по правам админа.
