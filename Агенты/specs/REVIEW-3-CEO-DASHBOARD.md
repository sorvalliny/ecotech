## ЧТО ХОЧЕТ ВИДЕТЬ CEO UNIT

1. **Когда запуски в прод** — даты коммитов перед ТОПами, отклонение от сроков
2. **Ресурсы команды** — укомплектованы ли, есть ли блокеры по людям
3. **Критические препятствия** — что мешает попасть в срок
4. **Сигналы риска:**
   - Продукт без данных = халтурит, не исполняет поручение
   - Продукт висит между стадиями > 2 недель = застрял, нужны причины
   - Фейковые данные = нужна сверка PMO
5. **Данные для 1-1** с продуктовыми лидерами — картина + фидбек + решения

## ОБЯЗАТЕЛЬНЫЕ vs РЕКОМЕНДУЕМЫЕ АТРИБУТЫ

### ОБЯЗАТЕЛЬНЫЕ (вшиты в процесс, ОРБИТА оцифровывает)

| Атрибут | Где | Кто заполняет | SLA |
|---------|-----|--------------|-----|
| Название + описание | Карточка продукта | PM | При создании |
| Текущая Орбита (Gate) | Карточка продукта | PMO подтверждает | При Gate Review |
| Статус (on-track / at-risk / blocked) | Карточка продукта | PM | Еженедельно |
| Владелец (PM) | Карточка продукта | Admin | При создании |
| Дата следующего Gate | Карточка продукта | PM + PMO | После Gate Review |
| Дата планируемого запуска | Карточка продукта | PM | При переходе на О·2+ |
| Размер команды | Карточка продукта | PM | Ежемесячно |
| ≥1 цель с KR | Цели | PM | В течение 2 нед после Gate 0 |
| ≥1 инициатива | Инициативы | PM | В течение 1 нед после Gate 0 |
| RICE-оценка на инициативах | Инициативы | PM | При создании |
| Gate-чеклист прогресс | Gate-чеклист | PM | До запроса Gate Review |
| Последнее обновление < 2 нед | Changelog | Авто | Автоматически |

### РЕКОМЕНДУЕМЫЕ (полезно, но не блокирует)

| Атрибут | Где |
|---------|-----|
| KPI метрики (MAU, Retention, NPS) | Карточка продукта |
| Трекер задач URL (YouTrack) | Карточка продукта |
| Конкурентный анализ | Бриф / шаблоны |
| P&L модель | Strategy (для О·3+) |

## СИГНАЛЫ РИСКА ДЛЯ CEO DASHBOARD

### Красные флаги (автоматические)

| Сигнал | Условие | Действие |
|--------|---------|----------|
| 🔴 Нет данных | Продукт без целей ИЛИ без инициатив | «Продукт не исполняет поручение» |
| 🔴 Застрял на стадии | lastUpdated > 14 дней | «Нет прогресса N дней. Причина?» |
| 🔴 At Risk / Blocked | status === 'at-risk' или 'blocked' | Подсветить + причина |
| 🟡 Нет RICE | Инициативы без r/i/c/e | «Приоритеты не оценены» |
| 🟡 KR прогресс < 30% | current/target < 0.3 | «Цели под угрозой» |
| 🟡 Команда неполная | teamSize < requiredSize | «Ресурсный дефицит» |
| 🟡 Дата запуска просрочена | launchDate < today && gate < 3 | «Отклонение от сроков» |

### Сверка PMO (антифейк)

| Что проверяет PMO | Как |
|-------------------|-----|
| Статус продукта | PMO подтверждает на еженедельном синке |
| Метрики KPI | PMO сверяет с Amplitude/BI |
| Gate-чеклист | PMO валидирует перед Gate Review |
| Changelog | PMO проверяет регулярность обновлений |

Реализация:
- Поле `pmoVerified: { date, by, status }` на карточке продукта
- Бейдж «Сверено PMO [дата]» / «Не сверено > 7 дней»
- PMO кнопка «Подтвердить данные» на карточке

## НОВЫЕ ПОЛЯ В ПРОДУКТЕ (DATA CONTRACT v2)

```javascript
{
  // Существующие...

  // НОВЫЕ ОБЯЗАТЕЛЬНЫЕ
  teamSize: number,          // Размер команды
  requiredSize: number,      // Требуемый размер
  launchDate: string,        // Дата планируемого запуска (ISO)
  nextGateDate: string,      // Дата следующего Gate Review (ISO)
  blockers: string,          // Текущие блокеры (текст)

  // PMO СВЕРКА
  pmoVerified: {
    date: string,            // Дата последней сверки
    by: string,              // Кто сверял (userId)
    status: string           // 'verified' | 'issues' | 'pending'
  },

  // ОБЯЗАТЕЛЬНОСТЬ
  compliance: {              // Автоматически рассчитывается
    hasGoals: boolean,
    hasInitiatives: boolean,
    hasRICE: boolean,
    gateChecklistProgress: number, // 0-100%
    lastUpdatedDays: number,
    score: number            // 0-100 (% заполненности обязательных)
  }
}
```

## ПЛАН РЕАЛИЗАЦИИ

### Шаг 1: Добавить поля в продукт (tracker.html + DATA CONTRACT)
- teamSize, requiredSize, launchDate, nextGateDate, blockers
- pmoVerified object
- compliance auto-calc

### Шаг 2: Dashboard CEO — полный рефакторинг
- Убрать текущий MVP
- Секция «Сроки и запуски»: таблица продуктов × даты
- Секция «Сигналы риска»: красные/жёлтые флаги
- Секция «Сверка PMO»: что проверено, что нет
- Секция «1-1 prep»: карточка продукта для обсуждения

### Шаг 3: Compliance score на карточке продукта
- Прогресс-бар «Заполненность: 73%»
- Красный если < 50%, жёлтый 50-80%, зелёный > 80%

### Шаг 4: PMO кнопка «Подтвердить данные»
- Доступна для admin/pmo
- Ставит pmoVerified.date = today, .status = 'verified'
- Бейдж на карточке
