# B-006 | Интерактивный Gate-чеклист
## Агент: Разработчик платформы | Приоритет: P1

---

## Цель

Создать `Продукту/gate-checklist.html` — интерактивный чеклист прохождения Gate с прогресс-баром и ссылками на шаблоны.

---

## Функциональные требования

### Навигация по Gate
- Табы: Gate 0 | Gate 1 | Gate 2 | Gate 3 | Gate 4 | Gate 5 | Gate 6
- Источник данных для критериев: stages.js + хардкод расширенных чеклистов
- Каждый таб показывает: название Орбиты, описание перехода, кто решает, SLA

### Чеклист критериев
- Разделение на **обязательные** (блокирующие) и **желательные**
- Чекбоксы с сохранением состояния в localStorage (ключ: `ECOTECH_GATE_CHECKLIST`)
- Формат хранения: `{ productId: { gate0: [true, false, true, ...], gate1: [...] } }`

### Gate 0 → О·1 Разведка (пример)
**Обязательные:**
- [ ] Проблема описана одним предложением и понятна заказчику
- [ ] Целевая аудитория определена (сегмент + объём)
- [ ] Связь с OKR компании или департамента подтверждена
- [ ] Нет дублей в реестре продуктов
- [ ] Команда найдена или путь к поиску согласован
- [ ] RICE-оценка проведена → [Открыть RICE-калькулятор](rice.html)

**Желательные:**
- [ ] Конкурентный обзор (3–5 аналогов)
- [ ] Первичная оценка ресурсов (T-shirt)

### Прогресс-бар
- Live-подсчёт: N из M обязательных выполнено
- Цвет: серый (0–49%) → жёлтый (50–99%) → зелёный (100%)
- Текст: «3 из 6 обязательных критериев» / «Все критерии выполнены»

### Ссылки на шаблоны
- Для каждого критерия — ссылка на релевантный шаблон из templates.html
- Формат: `[Шаблон T01 →](templates.html?t=T01)`

### Кнопка «Запросить Gate Review»
- Активна только при 100% обязательных
- Действие: mailto PMO с предзаполненной темой и телом
- Тема: `Gate [N] Review — [Продукт] — [дата]`
- Тело: список выполненных критериев + ссылки на артефакты

### Выбор продукта
- Dropdown в header: выбор продукта из ECOTECH_PRODUCTS
- Состояние чеклиста привязано к продукту
- Текущий Gate подсвечивается автоматически
