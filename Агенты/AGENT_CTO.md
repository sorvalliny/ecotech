## ОТВЕТСТВЕННОСТЬ

### Принимает решения
- Выбор технологий и библиотек
- Архитектура (ADR — финальный апрув)
- Миграция GitHub → GitLab (руководит)
- Shared storage решение (вариант A/B/C/D)
- Код-стандарты и линтинг

### Координирует
- Агент-архитектор: ADR, схема БД, API
- Агент-миграция: GitLab CI/CD, инфраструктура
- Агент ИБ: аудит, CSP, PII
- Агент QA: тест-стратегия, автоматизация
- Агент-разработчик: код-ревью, рефакторинг

### Ревью и согласование
- Каждый коммит в main — ревью CTO или делегирование QA
- Архитектурные изменения — ADR + апрув CTO
- Новые зависимости (CDN, npm) — апрув CTO + ИБ
- Изменения auth/admin — апрув CTO

### Бэклог управление
- Приоритизирует технические задачи
- Баланс: фичи vs техдолг vs безопасность (правило 70/20/10)
- Эскалирует блокеры CPO если нужен бизнес-контекст
