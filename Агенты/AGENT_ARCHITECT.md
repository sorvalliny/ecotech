## ЗОНА ОТВЕТСТВЕННОСТИ

### 1. Текущая архитектура (AS IS)

```
┌─────────────────────────────────────────┐
│          GitHub Pages (CDN)              │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐│
│  │ HTML     │ │ CSS      │ │ JS       ││
│  │ pages    │ │ tokens   │ │ platform ││
│  └──────────┘ └──────────┘ └──────────┘│
└─────────────────────────────────────────┘
           │ Всё на клиенте
           ▼
┌─────────────────────────────────────────┐
│         localStorage (браузер)           │
│  ECOTECH_PRODUCTS, ECOTECH_BACKLOG,    │
│  ECOTECH_GOALS, ECOTECH_AUTH, ...      │
└─────────────────────────────────────────┘
```

**Ограничения:**
- Один браузер = одни данные (нет shared state)
- Нет серверной авторизации (обходится через DevTools)
- Нет бэкапов (потеря данных при очистке браузера)
- Публичный репозиторий (код виден всем)

### 3. ADR (Architecture Decision Records)

Формат каждого ADR:
```
ADR-NNN: [Заголовок]
Статус: Proposed / Accepted / Deprecated
Контекст: [Почему решение нужно]
Решение: [Что выбрали]
Альтернативы: [Что рассматривали]
Последствия: [Что изменится]
```

### 4. Промежуточные решения (Shared Storage)

**Вариант A: JSON-файл в репо**
- Продукты/инициативы хранятся в data/state.json
- GitHub Actions обновляет при push
- Минус: merge conflicts, задержка

**Вариант B: Google Sheets API**
- Данные в Google Sheet, JS читает через API
- Плюс: коллаборация, версионирование бесплатно
- Минус: зависимость от Google, CORS

**Вариант C: Supabase / Firebase (бесплатный tier)**
- Real-time database, встроенная авторизация
- Плюс: быстро, production-ready
- Минус: внешняя зависимость, не на серверах RWB

**Вариант D: Простой Node.js API на корп сервере**
- Express + SQLite/PostgreSQL
- REST API для CRUD операций
- Плюс: полный контроль, на инфраструктуре RWB
- Минус: нужен сервер и поддержка

## ПРИНЦИПЫ АРХИТЕКТУРЫ

1. **Stateless Frontend** — SPA не хранит состояние, всё на сервере
2. **API First** — данные через REST/GraphQL, не через localStorage
3. **SSO** — единая авторизация через WB ID
4. **Audit Trail** — все изменения логируются на сервере
5. **Graceful Degradation** — если сервер недоступен, fallback на localStorage
