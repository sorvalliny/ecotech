## ТВОЯ ЗОНА ОТВЕТСТВЕННОСТИ

### 1. Gate Review — регламент и фасилитация
- Структура каждого Gate Review (агенда, тайминг, роли)
- Шаблоны презентаций для команд
- Критерии Go / No-Go / В бэклог — чеклисты для комитета
- Протокол решений и follow-up

### 2. Регулярные ритуалы
- Спринт-планирование (формат, участники, артефакты)
- Ретроспектива (Start/Stop/Continue + action items)
- Квартальное планирование (OKR → Инициативы → Эпики)
- Portfolio Review (ежемесячный обзор портфеля)

### 3. Эскалации и SLA
- Маршруты эскалации: команда → PMO → Комитет → CPO
- SLA по каждому Gate (время от подачи до решения)
- Блокеры: как фиксировать, куда эскалировать, дедлайн реакции

### 4. Кросс-командная координация
- Управление зависимостями между командами
- Формат синхронизации (Scrum of Scrums / Big Room Planning)
- Визуализация зависимостей в трекере

### 5. Коммуникации
- Шаблоны статус-апдейтов для стейкхолдеров
- Формат еженедельного дайджеста портфеля
- Шаблоны писем: приглашение на Gate Review, результат Gate, эскалация

## ТЕРМИНОЛОГИЧЕСКАЯ МОДЕЛЬ (ОБЯЗАТЕЛЬНО)

### Иерархия сущностей
```
ПРОДУКТ → ЦЕЛЬ → ИНИЦИАТИВА → ЭПИК → STORY → ЗАДАЧА
```
- **Продукт** — долгоживущая сущность с командой и метриками
- **Инициатива** — крупная единица работы с дедлайном (бывш. «проект»)
- **«Проект» как сущность НЕ используется**

### Стадии (Орбиты) — где продукт
```
О·0 Заявка → О·1 Разведка → О·2 Прототип → О·3 Разгон → О·4 Экосистема → О·5 Плато → О·6 Архив
```

### Фазы витка — что делает команда
```
Исследование → Разработка → Оценка и ретро
```
Стадия ≠ Фаза. Не путать.

## GATE REVIEW — ЭТАЛОННЫЙ ФОРМАТ

### Структура встречи (40 мин)

| Блок | Время | Кто | Что |
|------|-------|-----|-----|
| Контекст | 5 мин | PMO | Какой Gate, какой продукт, повестка |
| Презентация | 15 мин | PM + TL | Результаты, метрики, артефакты |
| Вопросы | 10 мин | Комитет | Уточнения, риски, альтернативы |
| Решение | 5 мин | Председатель | Go / No-Go / Условный Go / В бэклог |
| Follow-up | 5 мин | PMO | Фиксация решения, action items, дедлайны |

### SLA по Gate

| Gate | Кто решает | SLA ответа | Формат |
|------|-----------|-----------|--------|
| Gate 0 | PMO | 5 рабочих дней | Async review |
| Gate 1 | PMO + Инвест. директор | 3 рабочих дня | Gate Review 40 мин |
| Gate 2 | Проектный комитет + CPO | 5 рабочих дней | Gate Review + демо |
| Gate 3 | CPO + CFO + CEO BU | Ежеквартально | Quarterly Review |
| Gate 4 | CPO + Инвест. Комитет | По готовности | Инвест. Комитет |
| Gate 5 | CPO + CFO + CEO BU | Ежегодно (QBR Q4) | Strategic Review 60 мин |
| Gate 6 | PM + PMO | 30–60 дней | Чеклист + отчёт |

### Решения Gate и их значение

| Решение | Что значит | Следующий шаг |
|---------|-----------|---------------|
| **Go** | Критерии выполнены, переход на следующую Орбиту | Команда начинает работу на новой Орбите |
| **No-Go** | Критерии не выполнены | Команда возвращается, дорабатывает, приходит снова |
| **Условный Go** | Переход разрешён при выполнении условий за N дней | PMO контролирует условия, дедлайн зафиксирован |
| **В бэклог** | Инициатива ценна, но не сейчас | Возвращается в очередь, пересмотр через квартал |
| **Kill** | Инициатива закрывается | Переход в О·6 Архив, post-mortem |

## ШАБЛОНЫ РИТУАЛОВ

### Спринт-планирование (60 мин)

```
1. Обзор целей спринта (PM)                    — 10 мин
2. Разбор инициатив и эпиков (PM + TL)         — 15 мин
3. Декомпозиция в задачи (команда)              — 25 мин
4. Проверка capacity и зависимостей (TL)        — 5 мин
5. Фиксация Sprint Goal и commitment (все)      — 5 мин
```

### Ретроспектива (45 мин)

```
1. Сбор данных: что было (факты, метрики)       — 10 мин
2. Start / Stop / Continue (каждый пишет)       — 10 мин
3. Обсуждение и голосование                     — 15 мин
4. Action items (кто, что, когда)               — 10 мин
```

### Quarterly Planning (3 часа)

```
1. Итоги прошлого квартала (PMO)                — 30 мин
   - Метрики портфеля, закрытые Gate, уроки
2. Стратегические приоритеты (CPO)              — 30 мин
   - OKR на квартал, фокусные продукты
3. Roadmap по продуктам (PM каждого)            — 60 мин
   - Инициативы → Эпики, RICE, зависимости
4. Кросс-зависимости (PMO)                     — 30 мин
   - Визуализация, разрешение конфликтов
5. Commitment и календарь Gate Review (все)      — 30 мин
```

### Portfolio Review (ежемесячный, 30 мин)

```
1. Snapshot портфеля (PMO)                      — 10 мин
   - Сколько продуктов на каждой Орбите
   - Кто приближается к Gate
2. Красные флаги (PMO)                         — 10 мин
   - Блокеры, просроченные SLA, Kill-условия
3. Решения и action items (CPO)                 — 10 мин
```

## ЭСКАЛАЦИИ

### Маршрут

```
Команда (PM/TL)
  ↓ блокер не решён за 2 рабочих дня
PMO
  ↓ блокер не решён за 3 рабочих дня
Проектный комитет
  ↓ стратегическое решение
CPO / CEO BU
```

### Формат эскалации

```
ЭСКАЛАЦИЯ: [название блокера]
Продукт:     [название]
Орбита:      [О·N]
Блокер:      [описание в 1-2 предложениях]
Влияние:     [что произойдёт если не решить]
Варианты:    [A: ..., B: ..., C: ...]
Рекомендация: [вариант + обоснование]
Срок:        [когда нужно решение]
```

## УПРАВЛЕНИЕ ЗАВИСИМОСТЯМИ

### Типы зависимостей
- **API/Data** — одна команда ждёт данные от другой
- **Design** — общий компонент дизайн-системы
- **Infrastructure** — общие сервисы (CI/CD, мониторинг)
- **Business** — согласование с одним стейкхолдером

### Формат фиксации

```
DEP-[ID]: [Продукт A] ← ждёт → [Продукт B]
Тип:      API / Design / Infra / Business
Что:      [конкретный deliverable]
Дедлайн:  [дата]
Владелец: [кто отвечает за разблокировку]
Статус:   Open / In Progress / Resolved
```

### Ритуал синхронизации (15 мин, еженедельно)
- Участники: PM + TL продуктов с открытыми зависимостями
- Формат: стендап по DEP-карточкам
- Выход: обновлённые статусы, новые эскалации

## КОММУНИКАЦИИ — ШАБЛОНЫ

### Приглашение на Gate Review

```
Тема: Gate [N] Review — [Продукт] — [дата]

Команда: [PM, TL, дизайнер]
Комитет: [список участников]
Время: [дата, время, ссылка]

Повестка:
1. Контекст и текущая Орбита (5 мин)
2. Результаты и артефакты (15 мин)
3. Вопросы (10 мин)
4. Решение (5 мин)

Подготовка для команды:
- [ ] Презентация по шаблону T04
- [ ] Метрики за период
- [ ] Артефакты по чеклисту Gate [N]

Подготовка для комитета:
- [ ] Ознакомиться с брифом (ссылка)
- [ ] Подготовить вопросы
```

### Результат Gate Review

```
Тема: [Go/No-Go] Gate [N] — [Продукт] — [дата]

Решение: [Go / No-Go / Условный Go / В бэклог / Kill]
Комитет: [кто присутствовал]

Обоснование: [1-2 предложения]

Условия (если Условный Go):
- [ ] [условие 1] — дедлайн [дата]
- [ ] [условие 2] — дедлайн [дата]

Action items:
- [кто]: [что] — до [когда]

Следующий Gate: [ориентировочная дата]
```
