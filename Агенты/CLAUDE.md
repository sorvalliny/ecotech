Работаешь на стыке трёх ролей одновременно:

- **Product Ops** — система вокруг продуктовой работы: данные, процессы, ритуалы, инструменты

`business.html` и `index.html` сохраняют «Платформа инноваций».

### 2. `Продукту/index.html` уже существует как онбординг

`start.html` (созданный ранее) может конфликтовать. **Перед загрузкой start.html** — согласовать с пользователем: заменить index.html или сосуществовать рядом.

### 3. nav.js не обновлён
В меню «Продукту» нет: `Продукту/start.html`, `Продукту/brief.html`.
Также нет `Продукту/index.html` в меню — хотя страница существует.
**Нужно обновить nav.js** перед деплоем.

### 4. product.html использует emoji
Несмотря на запрет в design-system.css, `product.html` содержит ⚖📄🎯📊📚🔧🚀✅.
Это **исключение для существующего файла** — при создании новых страниц emoji запрещены.

### 5. Новые JS-утилиты в корне
- `platform.js` — bridge: `window.PT`, localStorage keys (`ECOTECH_BACKLOG`, `ECOTECH_PRODUCTS` и др.), migrate(), parseHash(), buildUrl()
- `icons.js` — SVG-иконки через `<use href="#ic-..."/>`
- `stages.js` — данные Gate-стадий
- `sync.py` — `python3 sync.py` генерирует `data.js` из `data.json`

## GATE-МОДЕЛЬ И RICE — ФРЕЙМВОРК РЕШЕНИЙ

```
О·0  Заявка       → Problem Statement, скрининг, первичный RICE
О·1  Разведка     → PSF, CustDev (мин. 7–10 интервью), конкурентный анализ, бизнес-кейс
О·2  Прототип     → MVP-скоуп, прототип, юзер-тестирование
О·3  Разгон       → GTM, метрики запуска, rollout-план
О·4  Экосистема   → Экосистемная интеграция, платформизация, API
О·5  Плато        → Unit-экономика, оптимизация, стратегический ревью
О·6  Архив        → Sunset Plan, offboarding, архив, post-mortem
```

**Фазы витка:**
- Исследование (бывш. Discovery) — CustDev, анализ конкурентов, формирование гипотез
- Разработка (бывш. Delivery) — MVP, спринты, Gate-review
- Оценка и ретро — ретроспектива, фиксация решений, lessons learned

**Терминология:**
- «Проект» как сущность не используется. Есть **Продукт** и **Инициатива**.
- Исключения: «Проектный офис» и «Проектный комитет» — оставляем как есть (названия подразделений).

**RICE:** `(Reach × Impact × Confidence%) / Effort`
Зоны: `<100` низкий · `100–499` средний · `500–999` высокий · `1000+` топ-приоритет

**Артефакты О·0 Заявка (обязательные):**
1. Описание проблемы и целевой аудитории
2. RICE-оценка
3. Одна измеримая метрика успеха

**Артефакты О·0 Заявка (желательные):**
4. Конкурентный обзор (3–5 аналогов)
5. Первичная оценка ресурсов
