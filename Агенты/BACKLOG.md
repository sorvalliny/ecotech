## Как работать с бэклогом

1. **Методолог** создаёт задачи и спецификации
2. **Исполнитель** берёт задачу по ID, читает спецификацию
3. После выполнения — обновляет статус на DONE и коммитит
4. **Методолог** проводит аудит результата

**Статусы:** `TODO` → `IN_PROGRESS` → `REVIEW` → `DONE`
